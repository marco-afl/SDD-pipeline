## US-001 — Transferencia de dinero entre cuentas NovaPay

**Como** cliente bancario de NovaPay
**Quiero** enviar dinero desde mi cuenta hacia la cuenta de otro titular
**Para** realizar pagos o transferencias de forma segura y trazable

---

### Criterios de aceptación

- [ ] CA-1: El cliente puede iniciar una transferencia especificando cuenta origen, cuenta destino y monto.
- [ ] CA-2: El sistema descuenta el monto de la cuenta origen y lo acredita en la cuenta destino de forma atómica (ambas operaciones ocurren o ninguna).
- [ ] CA-3: La respuesta exitosa incluye: `transfer_id`, `status: "completed"`, `timestamp`, `amount`, `origin_account`, `destination_account`.
- [ ] CA-4: Cada transferencia exitosa queda registrada en el log de auditoría con: `transfer_id`, `origin_account`, `destination_account`, `amount`, `timestamp`, `initiated_by` (ID de cliente), `ip_address`.
- [ ] CA-5: El sistema soporta idempotencia mediante un campo `idempotency_key` enviado por el cliente; si se reenvía la misma key, devuelve la respuesta original sin procesar la transferencia dos veces.
- [ ] CA-6: Si la `idempotency_key` ya existe pero con parámetros distintos (monto o cuentas diferentes), el sistema devuelve error `409 Conflict`.

---

### Casos de error

| Escenario | HTTP Status | Código de error | Mensaje |
|-----------|-------------|-----------------|---------|
| Saldo insuficiente en cuenta origen | `422 Unprocessable Entity` | `INSUFFICIENT_FUNDS` | "El saldo disponible es insuficiente para completar la transferencia." |
| Cuenta origen no existe o no pertenece al cliente | `404 Not Found` | `ORIGIN_ACCOUNT_NOT_FOUND` | "La cuenta de origen no fue encontrada." |
| Cuenta destino no existe | `404 Not Found` | `DESTINATION_ACCOUNT_NOT_FOUND` | "La cuenta de destino no fue encontrada." |
| Monto negativo o igual a cero | `400 Bad Request` | `INVALID_AMOUNT` | "El monto debe ser mayor a cero." |
| Monto con más de 2 decimales | `400 Bad Request` | `INVALID_AMOUNT` | "El monto no puede tener más de 2 decimales." |
| Cuenta origen igual a cuenta destino | `400 Bad Request` | `SAME_ACCOUNT_TRANSFER` | "La cuenta de origen y destino no pueden ser la misma." |
| `idempotency_key` con parámetros distintos | `409 Conflict` | `IDEMPOTENCY_CONFLICT` | "La clave de idempotencia ya fue usada con parámetros distintos." |

Todos los errores devuelven el cuerpo:
```json
{
  "error_code": "INSUFFICIENT_FUNDS",
  "message": "El saldo disponible es insuficiente para completar la transferencia.",
  "transfer_id": null,
  "timestamp": "2026-06-09T14:30:00Z"
}
```

### Contrato de API (borrador)

```
POST /v1/transfers
```

**Request:**
```json
{
  "idempotency_key": "uuid-v4",
  "origin_account": "ACC-00123",
  "destination_account": "ACC-00456",
  "amount": 1500.00,
  "currency": "USD",
  "description": "Pago de renta julio"   // opcional
}
```

**Response 201 Created:**
```json
{
  "transfer_id": "TRF-20260609-00001",
  "status": "completed",
  "origin_account": "ACC-00123",
  "destination_account": "ACC-00456",
  "amount": 1500.00,
  "currency": "USD",
  "timestamp": "2026-06-09T14:30:00Z"
}
```

### Casos límite

- Monto con más de 2 decimales: se rechaza con `400 INVALID_AMOUNT`.
- Transferencias en moneda distinta a la de la cuenta: fuera de scope por ahora (ver sección siguiente).
- Concurrencia: dos transferencias simultáneas desde la misma cuenta origen deben ser serializadas para no permitir sobregiro.
- Timeout de red durante la transacción: si la operación inició pero no completó, el estado debe quedar como `pending` y resolverse mediante proceso de reconciliación (fuera de scope de esta story).

---

### Fuera de scope

- Transferencias internacionales o multi-moneda.
- Notificaciones push / email al destinatario.
- Reversión (chargeback) de transferencias.
- Límites diarios o por transacción (se tratarán en story separada).
- Autenticación y autorización del cliente (se asume resuelta por la capa de API Gateway).

---

### Dependencias

- Servicio de cuentas: debe exponer consulta de saldo y bloqueo de fondos.
- Servicio de auditoría: debe aceptar eventos de transferencia para registro.
